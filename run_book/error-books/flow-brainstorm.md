# 错题本 — 五方脑暴/评审

> 适用场景：拉五方脑暴、议题评审、决策呈现

### 记录规则

- **条目结构**：❌ 错误做法 + ✅ 正确做法，整条控制在 **5 行以内**
- **❌/✅ 写法**：只写核心规则，禁止多层加粗嵌套（加粗最多一层）
- **`>` 注释**：只放归因路径字母和 postmortem 链接，禁止复述场景
- **复犯**：只追加频率标记（🟡×N / 🔴×N）
- **归属**：条目只写本范围内容，跨范围写到对应文件

### DEV-46 打包评审 + 评审缺背景 `🟢`

❌ 多个独立维度打包成一个议题评审；拉评审时不提供决策背景
✅ 不同维度拆开独立讨论；拉 agent 必须提供：痛点+竞品参考+用户已拍板方向+议题范围

### DEV-48 有存活 agent 却重新拉人 `🟢`（含 DEV-55）

❌ team 成员仍 idle 却新建 agent；多轮脑暴不 resume 上一轮 agent
✅ 切换议题：idle→SendMessage 复用；已 shutdown→才新建。同一议题多轮：用 `resume` 参数复用上一轮 agent

### DEV-49 议题结论未落盘纪要 + 未写入 MCP 记忆 `🟢`

❌ 议题拍板后跳过纪要文件，用户补充内容未同步到 IR，跨 session 丢失
✅ 拍板后三步缺一不可：① 创建纪要文件 ② 提炼决策到 IR ③ `add_message` 写入 MCP 记忆

### DEV-50 脑暴被锚定，无人挑战前提假设 `🟢`（含 DEV-52）

❌ 五方全程在 IR 框架/prompt 代码架构内优化，无人质疑前提假设
✅ 脑暴第 1 轮必须包含前提挑战环节：每位专家先回答"前提假设是什么？成立吗？"。5/5 无人质疑→主持人追问一次。阶段 0 prompt 只给痛点+约束+核心问题（what/why），禁止塞实现细节

### DEV-51 向用户提交决策时违反呈现三原则 `🟡中频×2`

❌ 多个子决策混在一起；数字无推导来源；方案不带描述假设用户记得全文
✅ 自检三条：① 子决策独立呈现独立确认（复合决策先拆再按依赖序逐个提交）② 数字附推导来源 ③ 方案自带一句话描述
