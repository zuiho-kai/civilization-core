Continuous Agent Society Engine
一个持续运行的制度生成世界
1. 我们在做什么？

我们正在构建一个持续运行的虚拟社会。

这个社会由数百个自主 Agent 组成。
它不是回合制模拟器，不是强化学习沙盒，也不是脚本驱动的剧情系统。

它是一个：

实时运行、制度内生演化、无预设终局的社会引擎。

在这个世界中：

生产发生

交换发生

强制发生

组织形成

制度改变

权力集中又瓦解

这一切不由中心规则预设，而由结构条件推动。

2. 为什么要重新设计“社会模拟”？

当前大多数 AI 社会模拟存在三个结构性问题：

① 预设奖励函数

强化学习系统必须定义 reward。
这意味着系统从一开始就拥有“方向”。

根据演化博弈论（概念由 John Maynard Smith 提出），
当奖励结构固定，系统会向稳定策略收敛。

结果是：

行为趋同

结构单一

世界收敛

而真实社会长期处于非均衡状态。

② 同步回合更新

大多数模拟采用 tick 或回合驱动。

这会导致：

行为同步

群体共振

周期性崩溃

经济爆炸增长

复杂系统理论指出，非平衡系统通过局部扰动维持结构（参考 Ilya Prigogine 的耗散结构理论）。

真实社会是异步的。

③ 制度被写死

组织、国家、法律常作为预定义模块存在。

这意味着制度是“设计者意志”，
而不是结构演化结果。

制度经济学（Douglass North）指出：

制度源于重复互动与路径依赖。

我们选择让制度“生成”，而不是“创建”。

3. 我们的核心设计原则
原则一：连续时间

世界基于连续时间与事件驱动。

没有回合。
没有周期刷新。
只有发生的事件。

这借鉴了分布式系统的 Actor 模型（如 Erlang/OTP）。

意义：

避免同步塌缩

支持 500+ Agent 实时运行

允许局部连锁事件

原则二：极简行为原语

Agent 只能执行三类行为：

生产

交换

强制

所有复杂制度必须由这三类行为组合产生。

这一设计基于社会结构分析传统（如 Max Weber 对权力结构的界定）。

意义：

不预设制度模板

强制与合作共存

允许制度多样性

原则三：生产具有时间成本

生产占用时间并锁定资源。

这反映了经济学中“时间结构生产”的概念（Eugen Bohm-Bawerk）。

意义：

引入机会成本

促进协作

形成组织需求

原则四：局部网络社会

Agent 只拥有局部关系网络。

信息不会全局广播。

小世界网络理论（Duncan Watts）表明，
真实社会由局部连接构成。

意义：

制度并存

地区差异

文化分化

原则五：制度由压力触发变化

制度不会通过预设民主流程改变。

当效率下降、冲突上升、迁移加剧时，
规则会发生“突变”。

复杂系统理论（Stuart Kauffman）指出，
系统在扰动下会跃迁至新稳态。

意义：

制度非线性演化

不走单一路径

避免最终收敛

4. 这不是游戏规则系统

我们不定义：

哪种制度更先进

哪种制度更公平

哪种制度更文明

我们只定义：

资源守恒

时间成本

强制有代价

制度是结构结果。

如果一个系统：

不设终局

不设统一奖励

不预设制度

不同步更新

那么它就具备：

长期非均衡

结构竞争

权力循环

制度多样性

这使它不仅是娱乐产品，
也是一个可用于研究制度生成、组织行为与多智能体互动的实验平台。
