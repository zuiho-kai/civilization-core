Continuous Agent Society Engine
原型设计说明书 v1.1（含理论来源与设计动机）
1. 时间模型：连续时间 + 事件驱动
设计内容

世界基于：

连续时间

事件队列

无 tick

为什么这样设计？

固定 tick 会导致：

行为同步

周期性波动

策略收敛过快

人为时间颗粒

在大规模系统中，这会放大系统共振。

理论与现实来源

分布式系统实践
金融撮合系统采用事件驱动模型（如 Nasdaq 的撮合引擎）。
原因：市场是异步的，不能用回合。

Actor 模型
Erlang/OTP 使用消息驱动并发。
原因：减少全局同步锁。

复杂系统研究
Ilya Prigogine 提出耗散结构理论：
非平衡系统通过局部事件维持结构。

设计意义

避免策略同步塌缩

允许突发连锁事件

保证 500+ Agent 可扩展

2. 行为原语限制为三类（Produce / Exchange / Coerce）
设计内容

所有行为都必须分解为：

生产

交换

强制

为什么这样设计？

如果允许复杂行为模板：

制度被预定义

社会结构被预设

Agent 只是执行脚本

限制行为原语，制度才会涌现。

理论来源

政治经济学基础结构
Karl Marx 将社会结构归结为生产关系。
无论评价如何，其分析框架强调：
生产方式决定制度形态。

交换理论
George Homans 的社会交换理论：
社会关系源于交换互动。

强制权力模型
Max Weber 将国家定义为垄断合法暴力。

设计意义

制度不是写出来的

制度是行为稳定结构

强制与合作必须并存，否则世界会单调

3. 生产采用“时间占用模型”
设计内容

生产不是每 tick 增长。

而是：

占用时间

锁定资源

未来事件完成

为什么这样设计？

若生产是瞬时收益：

不存在机会成本

强制风险过低

经济会爆炸增长

现实中生产具有：

延迟

不确定性

中断风险

理论来源

交易成本理论
Ronald Coase 说明组织存在的原因是降低交易成本。

资本形成理论
Eugen Bohm-Bawerk 提出“时间结构生产”。

设计意义

引入时间稀缺性

形成组织协作需求

让制度产生经济基础

4. 网络局部化（无全局广播）
设计内容

每个 Agent 只维护局部 trust 网络。

信息不全局传播。

为什么这样设计？

全局信息会导致：

完全竞争

瞬时套利

策略收敛

现实社会是局部网络。

理论来源

小世界网络理论
Duncan Watts 研究小世界网络结构。

社会资本理论
Pierre Bourdieu 指出社会关系本身是资本。

设计意义

允许局部制度

允许文化差异

形成制度竞争

5. 组织自动生成（非手动创建）
设计内容

当网络密度与协作频率达到阈值：

自动生成组织。

为什么这样设计？

若组织可随意创建：

制度是玩家意志

不是结构产物

现实组织往往从高频协作中形成。

理论来源

演化博弈论
John Maynard Smith
稳定策略在重复互动中形成。

制度经济学
Douglass North
制度源于重复交易的路径依赖。

设计意义

组织不是工具

组织是结果

6. 制度通过“突变”而非投票
设计内容

制度变化由：

内部效率下降

成员迁移压力

冲突成本

触发 rule_mutation_event。

为什么这样设计？

投票机制会：

人为理性化

强制民主路径

现实制度变化往往是压力驱动。

理论来源

制度突变理论
Samuel Huntington
制度变迁常源于失序。

复杂适应系统理论
Stuart Kauffman
系统通过扰动跃迁至新稳态。

设计意义

制度非理性演化

防止线性优化

保持世界多样性

7. 引入强制成本递增
设计内容

组织规模越大：

监督成本平方增长

内部冲突概率上升

为什么这样设计？

否则会：

出现单一帝国

系统收敛

理论来源

帝国规模研究
Paul Kennedy 指出帝国过度扩张导致崩溃。

公共选择理论
James Buchanan
大规模集体决策效率下降。

设计意义

强制不会永远最优

保持动态平衡

8. 不定义“好制度”
设计内容

系统不内置：

道德

公平

文明进步方向

为什么这样设计？

一旦预设价值：

制度有终点

演化被锁定

理论来源

价值中立社会科学传统
Max Weber 提出“价值中立”。

演化视角
制度是适应结果，不是道德产物。

9. 这个设计的终极意义

它不是游戏引擎。

它是：

一个制度生成实验场。

它允许：

强制社会

合作社会

混合社会

崩溃循环

迁移潮

制度竞争

同时存在。
